About me

I am a highly motivated person who has some experiences in developing some basic websites applications as a front-end programmer, version management with Git.

Currently compiling my applications in order to end my college in 2020. I also am working on my thesis, and will end my college soon on July (as predicted).

But eventhough I tested the Mobile products manually, I still know how to use Appium to test any product which is developed in mobile version.

The first time I knew about software testing is when I was a Sophomore. There I learnt about the theory and a little practical thing of software testing. Hence, I started to use Selenium, Robot Framework,and Cucumber, to test any software according to the needs. And I have been using PostMan for API testing.

I like to challenge myself by learning new technologies. I have always been interested in SQA field because I have an eagle eye to find bugs of a software. And being an SQA is like playing puzzle for me, frustrating yet challenging. Frustrating because managing bugs and error of a software isn't easy, challenging because every time I find errors & bugs, I have to fix them all and it really challenges me!

I always keen to learn deep into the fundamentals of the languages, frameworks, and moving forward into web development with latest technologies required to work with it. I feel comfortable working in a team, and also independently.

e-Abs

A website application for teachers ...

Proyek ini merupakan proyek yang saya kerjakan bersama dengan tim saya, dikembangkan karena melihat adanya permintaan dari salah satu SMA di Laguboti, Toba Samosir yang menginginkan sebuah aplikasi berbasis website yang dapat membantu pekerjaan para guru yaitu mampu merekapitulasi data absensi siswa.

Lama pengerjaan proyek ini adalah 2 bulan, terhitung sejak November 2018 sampai Januari 2019. Proyek ini dikembangkan dengan menggunakan framework Spring. Kami menambahkan value proposition dari aplikasi ini yaitu berupa teknologi SMS-Gateway yang mampu memberikan laporan kepada orangtua siswa mengenai rekapitulasi data absensi anak yang bersangkutan setiap bulannya.

Teknologi tersebut memungkinkan adanya biaya SMS yaitu sebesar 200 rupiah/sms. Dengan adanya teknologi ini, dapat menghilangkan miss komunikasi yang mungkin terjadi antara guru dan para orang tua.

Keyword Extraction

Keyword extraction simulator developed by ...

Proyek ini merupakan proyek Tugas Akhir (TA) berupa penelitian terkait peningkatan akurasi untuk ekstraksi kata kunci dari dokumen berbahasa Indonesia.

Lama pengerjaan penelitian ini adalah 9 bulan, terhitung sejak Oktober 2019 sampai Juli 2020. Penelitian ini bertujuan untuk membuat suatu simulator yang dapat melakukan ekstraksi kata kunci secara otomatis.

Simulator dibangun dengan menggunakan bahasa pemrograman Python dengan menggunakan Jupyter Notebook. Adapun metode yang digunakan adalah kombinasi dari metodeFeature Selection dan metode Classification. Dari keseluruhan algoritma, dianalisis algoritma terbaik yang akan diimplementasi pada Tugas Akhir ini.
